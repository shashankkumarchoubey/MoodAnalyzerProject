## 📌 Project Overview
Most basic sentiment analyzers rely on simple binary word counting (Positive +1, Negative -1). This project aims to build a more nuanced, "out-of-the-box" mood analyzer using core, standard Java. 

Instead of binary outputs, this model categorizes text into four distinct emotional states: **Joy, Anger, Sadness, and Anxiety**. It doesn't just read *what* the user types, but *how* they type it, utilizing typography, punctuation context, and grammatical negation to calculate emotional intensity.

It also features a continuous interactive "rant" loop and a persistent Dashboard that saves the user's emotional profile across sessions using File I/O.

---

## 🚀 Key Features & Logic

### 1. Nuanced Emotion Mapping ($O(1)$ Complexity)
The vocabulary is managed using a Java `HashMap` (`EmotionDictionary.java`). This ensures that word lookups happen in **O(1) constant time**, making the text processing highly efficient compared to nested iterations.

### 2. Contextual Typography & Punctuation Multipliers
The analyzer looks beyond raw words:
* **ALL CAPS Multiplier:** Shouting a word (e.g., "LOVE" vs "love") automatically doubles its emotional weight.
* **High-Energy Punctuation:** Exclamation marks (`!!!`) act as multipliers for high-energy emotions (Joy/Anger).
* **Low-Energy Punctuation:** Ellipses (`...`) act as multipliers for low-energy, passive emotions (Sadness/Anxiety).

### 3. Smart Negation Checker
The algorithm checks the context of the preceding words. If a negator (e.g., "not", "never", "cant") is detected, it logically flips the emotion. For example, "not happy" converts the score from *Joy* to *Sadness* rather than mistakenly incrementing the Joy counter.

### 4. Interactive Rant Mode & Minimum Word Restriction
The `Main.java` class features a continuous `while(true)` loop equipped with a `Scanner`. To ensure the natural language processing has enough context to analyze (like checking for negations and punctuation), the system enforces a strict **6-word minimum limit** per entry.

### 5. Persistent Dashboard & Smart Directory Handling
The application tracks the user's cumulative emotional state. 
* Uses standard Java `FileWriter` and `File` classes to save data to a `user_mood_data.txt` file.
* **Smart Environment Detection:** The `MoodDashboard` dynamically checks the `user.dir` property. Whether the code is executed from the root directory via terminal or from within the `src` folder via an IDE, it automatically creates the `output` directory in the correct, root-level location to prevent `FileNotFound` exceptions.

## 💻 How to Run the Project

1. Open your terminal or command prompt.

2. Clone the repository to your local machine:
   ```bash
   git clone https://github.com/shashankkumarchoubey/MoodAnalyzerProject.git
   ```
3. Navigate to the `src` folder of the project:

    ```bash
    cd MoodAnalyzerProject/src
    ```

4. Compile all Java files:

    ```bash
    javac *.java
    ```

5. Run the Main application:

    ```bash
    java Main
    ```

**Start typing to see your real-time emotional analysis!**
